# tushare-data boundary trial summary

## Date
2026-03-23

## Scope
本总结记录 `tushare-data` 第三组真实测试（边界 / 模糊 / 失败题）的结果，重点验证 skill 在非主链场景下是否仍能保持清晰、克制、可解释的行为。

## Cases Covered
本轮真实测试覆盖：
1. 歧义标的：`帮我看下中航最近怎么样`
2. 模糊导出：`给我导一份数据`
3. 非交易日空表：`帮我查一个非交易日的日线数据`
4. 权限不足接口：`帮我梳理下最近的重要公告`
5. 超短高频问法：`今天最强板块`

## Current State
第三组测试结果说明：
- `tushare-data` 现在不只是主链任务可用，边界行为也开始成熟
- 它已经能在真实环境里正确区分：
  - 该澄清的时候澄清
  - 该说明空结果的时候说明空结果
  - 该承认权限不足的时候承认权限不足
  - 该把超短句补全成真实 workflow 的时候补全 workflow

## What Worked Well
### 1. 歧义标的不再盲猜
真实测试中，“中航”检索出了多个真实候选标的，证明：
- 这是高歧义请求
- 正确策略是最小澄清，而不是直接猜一个股票

### 2. 模糊导出能正确进入澄清 / dry-run 路线
“给我导一份数据”没有被误当成可直接执行的明确任务，而是被识别为：
- 需要澄清导出对象
- 需要澄清时间范围
- 需要澄清格式
- 如用户只想先看方案，应进入 dry-run

### 3. 非交易日空表得到正确解释
真实测试中，`2026-03-22` 被交易日历识别为非交易日，对应日线数据为 0 条。
这说明：
- 空表解释逻辑真实成立
- skill 没有把空表误报成“接口失败”

### 4. 权限不足接口能正确归因
公告接口 `anns_d` 再次命中权限不足，验证了：
- 这是 token / 权限问题
- 不是 skill / 代码问题
- 正确行为应是显式说明权限边界，而不是盲重试或胡乱改参数

### 5. 超短高频问法可以真实落地
“今天最强板块”这类极短请求，已经能在真实数据下落地为：
- 单日板块强度排行
- 可解释的结果输出

这说明 skill 已具备把超短句补全为完整研究任务的能力。

## Stable Rules Confirmed By Boundary Trial
以下规则经第三组真实测试后，可视为进一步确认：

- 歧义简称默认做最小澄清
- 模糊导出默认先澄清或进入 dry-run
- 空表不直接等同于失败，要优先检查是否非交易日 / 无数据 / 权限不足
- 权限失败必须明确归因为 token / 接口访问边界
- 超短高频问法应优先补全为高频 workflow，而不是要求用户重写问题

## What This Means
边界测试通过意味着：
`tushare-data` 现在不仅能完成“正常问题”，还开始具备：
- 克制
- 解释能力
- 边界感
- 失败归因能力

这比单纯跑通主链接口更重要，因为它说明 skill 已经接近真实长期使用所需的行为质量。

## Remaining Improvement Space
尽管第三组整体表现良好，后续仍可继续加强：
- 对歧义标的增加更优雅的候选展示格式
- 对模糊导出补更清晰的默认澄清模板
- 对权限不足接口补更明确的替代路线建议
- 对超短句补更多高频口语样例

## One-Line Summary
`tushare-data` 第三组边界测试通过，说明该 skill 已不仅主链可用，边界场景下也开始具备成熟 skill 所需要的克制、澄清、解释与失败归因能力。
