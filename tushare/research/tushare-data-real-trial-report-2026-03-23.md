## Problems Found
### Problem 1 — 财务趋势 workflow 不够贴题
在“看下比亚迪最近8个季度营收和净利润趋势”这题中，当前实际执行更偏向：
- ROE
- 毛利率
- 净利率

虽然结果不是错的，但与题目要求的“营收 + 净利润趋势”并不完全对齐。

#### Current issue
财务趋势类任务当前更容易优先走 `fina_indicator`，而不是优先走 `income`。

#### Fix direction
- 对“营收 / 净利润趋势”类请求，应优先拉 `income`
- 再用 `fina_indicator` 做补充质量解释
- 输出时应明确区分：营收、归母净利润、净利率、ROE 等不同层次指标

### Problem 2 — 北向资金汇总逻辑需要校验
在“北向资金最近在买什么”这题中，链路成功执行，但汇总结果中 `net_amount` 出现异常为 0 的现象。

#### Current issue
- 资金链路能跑
- 但聚合后的解释可信度需要二次确认

#### Fix direction
- 检查 `hsgt_top10` 字段语义与聚合逻辑
- 必要时先输出原始明细，再做汇总总结
- 若某字段在该接口中并不适合作为累计净买入口径，应改用更稳妥的解释方式

### Problem 3 — 筛选任务更像候选池生成，而非最终研究结论
在“帮我筛一批高ROE低负债的股票”这题中，筛选流程真实可跑，但结果更适合视为候选池，而不是最终建议。

#### Current issue
- 只凭 ROE + 负债率容易筛出高成长高估值标的
- 缺少估值、行业、市值、样本范围等进一步约束

#### Fix direction
- 默认增加更多筛选语义约束
- 明确说明“候选池”与“最终判断”的区别
- 输出模板中增加风险提示与样本范围说明

### Problem 4 — 导出任务目前是样本化成功，不是全量闭环
在“把沪深300成分股近两年日线导成CSV”这题中，真实导出已成功，但采用了样本化策略：
- 先用沪深300最新成分中的前 10 只股票做真实导出试运行

#### Current issue
- 导出路径与 CSV 生成已验证成功
- 但完整业务目标（全量 HS300 成分股近两年日线）还未完全执行

#### Fix direction
- 后续做全量版时，应补充分段策略、耗时说明、文件体量说明、元信息记录
- skill 在导出类任务中应区分“样本试跑”与“完整导出”

## Stable Decisions From Trial
以下结论在本轮真实试运行后可以视为稳定：

- `tushare-data` 已进入真实可用阶段，不再只是结构化说明 skill
- 主干任务链路已经足够支持继续迭代，而不是推倒重来
- 当前最有价值的工作不再是继续大幅补文档，而是“真实题目 → 抓问题 → 小修 workflow → 再验证”
- FAQ、playbooks、output templates 的方向已被真实环境验证为正确
- 后续优化应优先修具体 workflow 质量问题，而不是回到“继续堆接口文档”

## Superseded Assumptions
以下旧判断已被本轮真实试运行推翻：

- “这个 skill 可能只是文档看起来完整，真实跑不起来”
- “只有单接口查询才可能跑通，组合型任务大概率不稳”
- “导出类任务可能停留在纸面设计”

本轮真实结果显示：这些旧担忧已不再成立。

## Current Governing Constraints
后续继续迭代时，应遵守这些当前约束：

- 优先修真实任务中暴露出来的问题，不要盲目扩写文档
- 所有优化都应围绕“任务结果是否更贴题、更可信、更可交付”
- 涉及权限接口时，要明确区分 skill 问题和 token 权限问题
- 导出类任务必须清楚说明是样本试跑还是完整导出
- 筛选类任务默认应强调“候选池”而不是过度包装成结论
